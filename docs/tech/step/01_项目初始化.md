# 项目初始化

## 目标
创建 FastAPI Admin Pro 项目的基础架构和目录结构。

## 实施步骤

### 1. 创建项目目录结构
```bash
# 主要应用目录
mkdir -p app/{api/v1/{system,infra,ai},core,models/{system},schemas/{system},services/{system},middleware,utils,plugins/{mysql,postgresql,mongodb,redis,elasticsearch,milvus,kafka,oss,scheduler,ai,dataperm,dict,operlog}}

# 辅助目录
mkdir -p {tests,alembic,docs/{tech/{step,api,architecture},sql},scripts,docker}
```

### 2. 配置项目依赖
- 更新 `pyproject.toml` 使用现代化的项目配置
- 创建 `requirements.txt` 作为传统依赖管理
- 配置开发工具（ruff、mypy、pytest）

### 3. 项目结构设计
采用分层架构：
- **API 层**: 处理 HTTP 请求和响应
- **Service 层**: 业务逻辑处理
- **Model 层**: 数据模型定义
- **Schema 层**: 数据验证和序列化
- **Plugin 层**: 可扩展功能模块

### 4. 技术选型
- **Web 框架**: FastAPI 0.109+
- **ORM**: SQLAlchemy 2.0（异步支持）
- **数据验证**: Pydantic V2
- **配置管理**: Pydantic Settings
- **日志**: Loguru
- **测试**: pytest + httpx
